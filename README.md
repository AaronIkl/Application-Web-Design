## Guía de Comandos Git
Comandos básicos utilizados para el control de versiones:

* **Ver estado del repositorio:** `git status` (Muestra qué archivos han sido modificados o están listos para agregarse).
* **Agregar archivos:**
    * Individual: `git add nombre_del_archivo`
    * Global (todos): `git add .`
* **Commit con comentarios:** `git commit -m "Descripción clara del cambio"`
* **Subir cambios al remoto:** `git push origin nombre_de_la_rama`
* **Gestión de ramas:**
    * Crear rama: `git branch nombre_nueva_rama`
    * Ver ramas: `git branch`
    * Borrar rama: `git branch -d nombre_de_la_rama`
* **Regresar a un commit específico:** `git checkout <id_del_commit>` o `git reset --hard <id_del_commit>`

## Opciones de Etiquetado en Markdown

1. **Encabezados:** Se usa el símbolo `#`.
2. **Énfasis:** *Cursiva* con `*texto*` y **Negrita** con `**texto**`.
3. **Listas:** * No ordenadas: Usando guiones `-` o asteriscos `*`.
    * Ordenadas: Usando números `1.`, `2.`, etc.
4. **Código:** Para bloques de código se usan tres comillas invertidas ` ``` `.
5. **Enlaces:** `[Texto del enlace](URL)`.
6. **Imágenes:** `![Texto alternativo](URL de la imagen)`.
